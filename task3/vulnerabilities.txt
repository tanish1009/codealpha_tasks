Vulnerability 1: OS Command Injection (CWE-78)
Severity: CRITICAL
Description: The application uses Python's f-string string interpolation to stitch user inputs directly into
an operating system command template. Because formatting boundaries are not explicitly isolated, an
attacker can input shell punctuation sequences (like ;, &&, or ||) to execute arbitrary commands with
application runtime authorization.

Vulnerability 2: Use of a Inherently Dangerous Subshell API (CWE-789)
Severity: HIGH
Description: The codebase invokes os.system() to trigger the underlying networking utilities. This
API is highly discouraged because it implicitly spawns a live subshell framework (/bin/sh) to map the
execution parameters, which greatly exacerbates structural exploitation risks compared to direct binary
mapping mechanisms.

Vulnerability 3: Uncontrolled Resource Consumption / Thread Hangs (CWE-400)
Severity: MEDIUM
Description: The utility invokes system commands without managing network or time-based response
envelopes. An adversary can pass custom arguments or target destinations that stall the process
indefinitely, depleting active thread availability and leading to a structural Denial of Service (DoS).

Vulnerability 4: Sensitive Diagnostics Information Leakage (CWE-117)
Severity: LOW
Description: Raw console outputs and standard error stack traces generated by command failures are
passed directly back to the API client layer. This structural exposure facilitates reconnaissance and
mapping of underlying directory configurations by unauthorized entities.

Vulnerability 5: Missing Context-Specific Input Sanitization (CWE-20)
Severity: MEDIUM
Description: Parameter extraction relies completely on basic extraction from request.args without
implementing localized syntax validation patterns. The absence of alphanumeric character checks
allows structural parameters to hit execution sinks
